Authentication

User authentication endpoints (signup, login, token refresh, logout)

POST
/auth/signup
Register a new user
Create a new user account with validated credentials. Password must be strong (8+ chars with uppercase, lowercase, number, and special character)
Request Body
Parameter Type Description
usernamerequired string 3-20 characters, must start with letter or '_', contain only letters, numbers, '_' or '-'
emailrequired string Valid email address
passwordrequired string Minimum 8 characters with uppercase, lowercase, number, and special character
Example Request
{
  "username": "john_doe",
  "email": "john@example.com",
  "password": "SecurePass123!"
}
Responses
201 - Created
User created successfully
400 - Bad Request
User already exists or validation failed
500 - Internal Server Error
Server error occurred
POST
/auth/login
Login user
Authenticate user with email and password. Returns access token and refresh token (in httpOnly cookie)
Request Body
Parameter Type Description
emailrequired string User email address
passwordrequired string User password
Example Request
{
  "email": "john@example.com",
  "password": "SecurePass123!"
}
Example Response
{
  "ok": true,
  "message": "Login Successful",
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "username": "john_doe",
    "userID": "507f1f77bcf86cd799439011",
    "hasDigitalCard": false
  }
}
Responses
200 - OK
Login successful
400 - Bad Request
Wrong email or password
500 - Internal Server Error
Server error occurred
POST
/auth/refresh-token
Refresh access token
Generate a new access token using the refresh token stored in cookies. Refresh token is valid for 7 days
Responses
201 - Created
Access token refreshed successfully
401 - Unauthorized
No refresh token provided
403 - Forbidden
Invalid or expired refresh token
500 - Internal Server Error
Server error occurred
POST
/auth/logout
Logout user
Clear refresh token cookie and logout the user
Responses
200 - OK
Logout successful

Digital Card

Digital card management endpoints (create, read, update, delete)

GET
/api/digital-card/{userCardID}
Get digital card by user ID
Retrieve a published digital card using the user's ID. This is a public endpoint that requires no authentication
Path Parameters
Parameter Type Description
userCardIDrequired string MongoDB user ID
Responses
200 - OK
Digital card retrieved successfully
400 - Bad Request
Digital card does not exist
500 - Internal Server Error
Server error occurred
POST
/api/digital-card
Create a new digital card
Requires Authentication (Bearer Token)
Create a new digital card for the authenticated user. User can only have one digital card. Includes image upload to Cloudinary
Form Data Parameters
Parameter Type Description
imagerequired file Profile image file
namerequired string Full name (letters and Arabic characters only)
positionrequired string Job position or title
descriptionrequired string Bio or professional description (max 500 characters)
linksrequired string JSON string array of links
Example Links Format
[
  {
    "linkName": "email",
    "linkUrl": "john@example.com"
  },
  {
    "linkName": "linkedin",
    "linkUrl": "https://linkedin.com/in/john"
  }
]
Responses
201 - Created
Digital card created successfully
400 - Bad Request
Digital card already exists or validation failed
401 - Unauthorized
Missing or invalid access token
403 - Forbidden
Invalid or expired access token
500 - Internal Server Error
Server error occurred
PUT
/api/digital-card
Update digital card
Requires Authentication (Bearer Token)
Update the authenticated user's digital card. Image upload is optional; if not provided, existing image is retained
Form Data Parameters
Parameter Type Description
image file New profile image file (optional)
namerequired string Full name (letters and Arabic characters only)
positionrequired string Job position or title
descriptionrequired string Bio or professional description (max 500 characters)
linksrequired string JSON string array of links
Responses
200 - OK
Digital card updated successfully
400 - Bad Request
Digital card does not exist or validation failed
401 - Unauthorized
Missing or invalid access token
403 - Forbidden
Invalid or expired access token
500 - Internal Server Error
Server error occurred
DELETE
/api/digital-card
Delete digital card
Requires Authentication (Bearer Token)
Delete the authenticated user's digital card. Removes associated image from Cloudinary
Responses
200 - OK
Digital card deleted successfully
400 - Bad Request
Digital card does not exist
401 - Unauthorized
Missing or invalid access token
403 - Forbidden
Invalid or expired access token
500 - Internal Server Error
Server error occurred

üìã Data Models & Definitions

User

_id string

MongoDB unique identifier

username string required

Username (3-20 characters, starts with letter or '_', contains only letters, numbers, '_' or '-')

email string required

User email address

password string required

Password (minimum 8 characters with uppercase, lowercase, number, and special character)

hasDigitalCard boolean

Whether user has created a digital card (default: false)

DigitalCard

_id string

MongoDB unique identifier

userID string

Reference to the user who owns this card

name string required

Full name (letters and Arabic characters only)

position string required

Job position or title

description string required

Bio or professional description (max 500 characters)

imgSrc string required

URL of the profile image hosted on Cloudinary

imgId string required

Cloudinary image ID for deletion purposes

links array required

Array of social and contact links (CardLink objects)

CardLink

linkName string required

Type of social or contact link

Allowed values: whatsapp, phone number, email, linkedin, facebook, youtube, website, github, x, behance, cv
linkUrl string required

URL or contact information for the link

AuthToken

ok boolean

Success status

message string

Response message

accessToken string

JWT access token (valid for 15 minutes)

user object

User information (username, userID, hasDigitalCard)
